1 Python Enterprise CRUD Demo

This documentation walks through a small enterprise application demo.

The example focuses on a B2B discount maintenance scenario and demonstrates how Python applications are typically structured when they read from and write to a Microsoft SQL Server database.

Stack

Python 3.13, uv, Streamlit, pandas, sqlalchemy, pyodbc

Database

MSSQL Server with role-based views for data access

1.1 Learning Goals

By the end of this project, students should be able to:

  • set up a Python 3.13 project with uv
  • connect to Microsoft SQL Server with pyodbc, sqlalchemy, and pandas
  • implement typical CRUD-style flows in a Streamlit user interface
  • handle database errors and validation messages in a user-friendly way

1.2 Business Scenario

The application manages quantity-based B2B discounts. Sales staff need an internal tool to:

  • view existing discount rules
  • add a new discount for a selected customer
  • edit an existing discount if business conditions change

The SQL Server already exposes role-based views for this workflow, so the Python application does not work directly on the base table. This mirrors a common enterprise setup where database permissions are restricted through views.

1.6 What You Should Notice

  • app.py contains no raw SQL. The presentation layer only deals with user interaction.
  • discount_service.py contains no Streamlit code. It focuses on validation and business meaning.
  • discount_repo.py is the only place where SQL statements are written.
  • .env keeps credentials out of the source code.

This separation is not just a style preference. It makes the application easier to test, easier to explain in class, and easier to extend later.

1.7 Suggested Workflow

  1. Complete the operating-system-specific setup steps.
  2. Configure your .env file.
  3. Test the MSSQL connection with sql_example.py.
  4. Start the Streamlit application with uv run streamlit run app.py.
  5. Inspect how each UI action maps to the business and data layers.
